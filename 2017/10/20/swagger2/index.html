swagger 二 swagger常用注解

Posted on 2017-10-20 | In swagger | Visitors

本文主要介绍常用的注解,然后注重讲解平时开发中常用的几个注解,一般会影响我们接口的输入输出或者参数自动映射等。
官方wiki

@Api()

这个注解主要用在controller类上
用于类；表示标识这个类是swagger的资源
tags–表示说明
value–也是说明，可以使用tags替代

@ApiOperation

主要用在controller方法上

必选元素

限定符和类型 必需的元素和说明
java.lang.String value对这个操作的描述（即对方法的描述）

可选元素

限定符和类型 可选元素和说明
Authorization[] authorizations用户权限
java.lang.String consumes适合媒体类型
java.lang.String httpMethodhttp方法如 GET, PUT, POST, DELETE, PATCH, OPTIONS（如果不设置改属性，每个接口将会被设置为任一方法都可使用）
java.lang.String nickname这个接口的昵称
java.lang.String notes对这个方法的详细描述（最好要添加这个方法）
int position
java.lang.String produces内容类型
java.lang.String protocols 协议 http, https, ws, wss
java.lang.Class response返回vo类型
java.lang.String responseContainer
java.lang.String tags

@ApiParam()

主要用在controller方法、参数、字段说明上

@ApiModel()

主要用在输入输出的实体类上

@ApiModelProperty

主要用户实体类的属性上，起到描述的作用，让接口可读性更强

@ApiIgnore()

用于类，方法，方法参数 表示这个方法或者类被忽略

@ApiImplicitParam()

用在方法上，表示单独的请求参数 ：这个是平时用的最多的也是相对比较重要的一个api注解。

一般我们使用这个参数的时候,我们通常会指定paramType这个参数的参数值，方便swagger-ui生成参数的输入格式，方便我们调试使用

属性 取值 作用
paramType 查询参数类型
path 以地址的形式提交数据
query 直接跟参数完成自动映射赋值
body 以流的形式提交 仅支持POST
header 参数在request headers 里边提交
form 以form表单的形式提交 仅支持POST
dataType 参数的数据类型 只作为标志说明，并没有实际验证
Long
String
name 接收参数名
value 接收参数的意义描述
required 参数是否必填
true 必填
false 非必填
defaultValue 默认值

一般我们使用的时候 一般配置name，value，required,和paramType尤其是这个参数 ，它有五个值，默认的值是body，就是以流的形式提交且只支持Post方法；默认 paramType值为body即以流的形式提交。
